
# bank — модульное CLI-приложение на Dart

Консольное приложение для учета клиентов, сотрудников услуг и договоров банка с хранением данных в SQLite.

## Предметная область

Система банковского учета:

- клиенты (`Client`);
- услуги (`Service`);
- договоры (`Document`) с привязкой к клиенту, сотруднику и услуге:
- должности (`Poste`) с привязкой к сотруднику.

## Архитектура и структура папок

Проект разделен на модули по слоям:

```text
lib/
  salon.dart               # публичный экспорт
  src/
    domain/                # сущности и базовые интерфейсы
      identity.dart
      client.dart
      beauty_service.dart
      appointment.dart
    data/                  # SQLite и SQL-логика
      salon_database.dart
    cli/                   # меню и обработка ввода
      menu.dart
      input_helper.dart
bin/
  mmm.dart                 # точка входа (main)
test/
  mmm_test.dart            # базовые тесты
```

### Что вынесено в каждый слой и почему

- `domain`: описывает данные и преобразования (`toMap`/`fromMap`), не зависит от SQL и консоли.
- `data`: отвечает за хранение, создание таблиц и CRUD-запросы.
- `cli`: отвечает за взаимодействие с пользователем (`stdin`/`stdout`) и навигацию по меню.
- `bin`: только запуск приложения и закрытие ресурсов.

Такое разделение упрощает поддержку, тестирование и замену инфраструктуры (например, SQLite на другой способ хранения).

## Реализованная функциональность

- просмотр списка клиентов, услуг, договоров и сотрудников;
- добавление и удаление клиентов;
- добавление и удаление услуг;
- добавление и удаление сотруников;
- добавление и удаление договоров;
- добавление и удаление должностей;
- вывод всех таблиц БД в консоль.

## Валидация

Реализованы проверки:

- обязательный текстовый ввод: значение не пустое после `trim()`;
- числовые поля для услуги: цена и длительность должны быть корректными числами больше 0;
- дата/время записи: поддерживается `DateTime.parse()` и формат вида `YYYY-MM-DD HH:MM[:SS]`.

## Тесты

В проекте есть 3 обязательных типа тестов:

1. `toMap/fromMap` для сущности `Client`;
2. вставка и чтение клиента из временной SQLite;
3. валидный и невалидный парсинг даты/времени.

## Запуск

Установить зависимости:

```bash
dart pub get
```

Запустить приложение:

```bash
dart run
```

Запустить тесты:

```bash
dart test
```
